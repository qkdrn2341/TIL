## 오늘 배운것
오늘은 자바스크립트에서 요소에 이벤트를 등록하는 방법과 비동기 프로그래밍 개념을 학습하였다. 먼저 `addEventListener`를 활용하여 버튼 클릭과 같은 사용자 이벤트에 반응하도록 구현해보았다. 이벤트는 사용자의 행동에 따라 특정 기능을 실행할 수 있게 해주며, 콜백 함수를 통해 실제 동작을 정의한다는 점을 이해했다. 특히 submit 이벤트에서 `preventDefault()`를 사용해 기본 동작을 막고 조건에 따라 흐름을 제어하는 부분이 인상적이었다.

또한 비동기 프로그래밍의 개념을 배우며 자바스크립트가 순차적으로 실행되는 언어임에도 불구하고, 시간이 오래 걸리는 작업을 기다리지 않고 다음 코드를 먼저 실행할 수 있다는 점을 확인하였다. `setTimeout` 예제를 통해 실행 순서가 예상과 다르게 동작하는 것을 직접 확인하면서 비동기의 특징을 이해할 수 있었다. 이를 통해 사용자 경험을 해치지 않으면서 효율적으로 프로그램을 동작시키는 방식이라는 것을 알게 되었다.

오늘 학습을 통해 이벤트 처리로 사용자와 상호작용하는 방법과, 비동기 처리로 프로그램의 흐름을 효율적으로 구성하는 방법을 이해하게 되었으며, 두 개념 모두 실제 웹 개발에서 매우 중요하다는 것을 느꼈다.


## 느낀점
자바스크립트를 사용해서 웹 화면 구성 요소에 이벤트를 등록하고 삭제하고 이벤트 콜백함수를 구현하면서 이벤트 동작에 대해 이해하게 되었고 단일 스레드로 움직이는 자바 스크립트에서 비동기 프로그래밍을 통해 여러 이슈들에 대응할수있다고 느꼈다.

## 활용하기
`setTimeout`과 `setInterval`
서버 요청과 파일 업로드 타이머 등 시간이 오래 걸리는 작업에서 활용할수있다.
e.preventDefault(); 브라우저 기본동작을 방지 하는 함수 (우클릭시 컨텍스트 메뉴 노출 막기 등)
