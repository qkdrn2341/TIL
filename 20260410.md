## 1. 프로토타입 (Prototype)
- JS는 클래스 기반이 아니라 프로토타입 기반 상속
- 모든 객체는 [[Prototype]]으로 연결됨

const user = { activate: true };
const student = {};
student.__proto__ = user;

console.log(student.activate); // true

핵심
- 읽기: 프로토타입에서 찾음
- 쓰기: 자기 자신에 생성됨
- this는 호출한 객체 기준

---

## 2. 생성자 함수 + 프로토타입

function Student(name) {
  this.name = name;
}

Student.prototype = {
  activate: true
};

const s = new Student('홍길동');

핵심
- new → __proto__가 Student.prototype 연결
- 공통 기능은 prototype에 정의

---

## 3. 생성자 함수

function Student(name, age) {
  this.name = name;
  this.age = age;
}

규칙
- 첫 글자 대문자
- this는 생성될 객체

const s = new Student('판다', 5);

---

## 4. new 동작 원리

1. 빈 객체 생성
2. this 바인딩
3. 프로퍼티 추가
4. this 반환

---

## 5. new 없이 호출 문제

Student('몽키', 2);

- this → 전역 객체
- 전역 오염 발생

해결 (new.target)

function Dog(name) {
  if (!new.target) return new Dog(name);
  this.name = name;
}

## 7. 배열 조작 메서드

## 8. 탐색 메서드

indexOf('값')   // 인덱스 or -1

---

## 9. 배열 자르기

slice(start, end)   // 복사 (원본 유지)
splice(index, 개수) // 삭제/추가 (원본 변경)

---

## 10. 기타 메서드

join(',')   // 문자열 변환
reverse()   // 뒤집기 (원본 변경)

---

## 11. 반복 & 변환

forEach (반복, 반환 없음)
arr.forEach(v => console.log(v));

map (변환 → 새 배열)
const result = arr.map(v => v * 2);

filter (조건 → 새 배열)
const even = arr.filter(v => v % 2 === 0);

---

## 12. 고급 메서드

reduce (누적)
const sum = arr.reduce((acc, cur) => acc + cur, 0);

find (첫 번째 값)
arr.find(v => v > 3);

some / every
arr.some(v => v > 3);   // 하나라도 true
arr.every(v => v > 0);  // 전부 true

---

## 13. 정렬

arr.sort(); // 문자열 기준

arr.sort((a, b) => a - b); // 오름차순
arr.sort((a, b) => b - a); // 내림차순

---

## 핵심 요약

- JS 상속 = 프로토타입 기반
- __proto__로 객체 연결
- new → 객체 생성 + this 바인딩
- 생성자 함수는 prototype 활용이 핵심
- 배열 메서드는 원본 변경 여부 중요
- map / filter / reduce는 필수

## 느낀점
객체 기반 스크립트 언어인 자바 스크립트의 클래스를 배우고 여러가지 예제를 통해 내부적으로 어떻게 돌아가고 있는지 알게되는 시간이었다.
